# MIM Program Guides

Централизованное хранилище руководств, инструкций и методических материалов в формате Markdown для экосистемы AISYSTANT.

## Назначение репозитория

Этот репозиторий служит **основным источником знаний** для:
- **ДЗ-чекера** (Проверяльщика ДЗ) — ИИ-ассистента для автоматической проверки домашних заданий
- **MCP-сервера контента** — предоставляет структурированный доступ к материалам
- **Статической документации** — генерация сайта через MkDocs/Docusaurus

## Структура репозитория

```
guides/
├── programs/                    # Корневая папка программ
│   ├── 1-personal/             # Программа 1: Личное развитие
│   │   ├── 1.1-self-development/   # Руководство: Саморазвитие
│   │   │   ├── index.md            # Оглавление руководства
│   │   │   ├── 01-section/         # Раздел 01
│   │   │   │   ├── 01-subsection.md    # Подраздел (инструкция)
│   │   │   │   ├── 02-table.table.yaml # Таблица моделирования
│   │   │   │   └── assets/             # Изображения раздела
│   │   │   ├── tables/             # Сквозные таблицы руководства
│   │   │   ├── questions/          # Вопросы и кейсы
│   │   │   ├── _check/             # Рубрики для проверки ДЗ
│   │   │   ├── _meta/              # Метаданные руководства
│   │   │   └── assets/             # Общие изображения
│   │   └── ...
│   ├── 2-work/                 # Программа 2: Рабочее развитие
│   └── 3-research/             # Программа 3: Исследовательское
├── _templates/                 # Шаблоны файлов
├── _meta/                      # Глобальные метаданные
├── assets/                     # Общие ресурсы
└── manifest.json               # Манифест репозитория
```

## Система кодирования

### Логические коды (для ссылок)
- `P.G.S.SS` — Программа.Руководство.Раздел.Подраздел
- Пример: `1.1.01.02` — Программа 1, Руководство 1, Раздел 01, Подраздел 02

### Физические имена (для файловой системы)
- Формат: `NN-slug-name.extension`
- Пример: `01-attention/02-focus.md`

## Типы файлов

| Тип | Суффикс | Описание |
|-----|---------|----------|
| Инструкция | `.md` | Текстовое руководство в Markdown |
| Таблица | `.table.yaml` | Таблица моделирования с JSON Schema |
| Вопрос | `.question.yaml` | Вопрос или кейс для рефлексии |
| Рубрика | `.rubric.yaml` | Критерии оценки для ДЗ-чекера |
| Изображение | `.png/.jpg/.svg` | Иллюстрации в папках `assets/` |

## YAML Frontmatter

Каждый `.md` файл должен начинаться с YAML frontmatter:

```yaml
---
id: "1.1.01.02"
title: "Название подраздела"
description: "Краткое описание содержания"
order: 2
guide_ref: "1.1"
section_ref: "1.1.01"
tags: ["внимание", "практика"]
---
```

## Быстрый старт

1. Склонируйте репозиторий
2. Используйте шаблоны из `_templates/` для новых файлов
3. Следуйте правилам именования и структуры
4. Заполняйте YAML frontmatter для каждого файла
