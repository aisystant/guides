# MIM Program Guides

Централизованное хранилище руководств, инструкций и методических материалов в формате Markdown для экосистемы AISYSTANT.

## Назначение репозитория

Этот репозиторий служит **основным источником знаний** для:
- **ДЗ-чекера** (Проверяльщика ДЗ) — ИИ-ассистента для автоматической проверки домашних заданий
- **MCP-сервера контента** — предоставляет структурированный доступ к материалам
- **Статической документации** — генерация сайта через MkDocs/Docusaurus

## Структура репозитория

```
guides/
├── programs/                        # Корневая папка программ
│   ├── 1-personal/                  # Программа 1: Личное развитие
│   │   ├── 1.1-self-development/    # Руководство 1.1
│   │   │   ├── index.md             # Оглавление руководства
│   │   │   ├── 01-section/          # Раздел 01
│   │   │   │   ├── 01-subsection.md     # Подраздел (текст)
│   │   │   │   ├── 02-table.table.yaml  # Таблица моделирования
│   │   │   │   ├── 03-case.case.yaml    # Кейс (выбор ответа)
│   │   │   │   ├── tables/              # Общие таблицы раздела
│   │   │   │   ├── questions/           # Контрольные вопросы раздела
│   │   │   │   └── assets/              # Медиа раздела
│   │   │   ├── tables/              # Общие таблицы руководства
│   │   │   ├── questions/           # Общие вопросы руководства
│   │   │   └── _meta/               # Метаданные руководства
│   │   └── ...
│   ├── 2-work/                      # Программа 2: Рабочее развитие
│   └── 3-research/                  # Программа 3: Исследовательское
├── _templates/                      # Шаблоны файлов
├── _sources/                        # Исходные материалы для обработки
└── manifest.json                    # Манифест репозитория
```

## Система кодирования

### Нумерация объектов

**Inline** (в потоке контента):
- Формат: `P.G.SS.NN` — Программа.Руководство.Раздел.Номер
- Пример: `1.1.01.02` — Программа 1, Руководство 1, Раздел 01, Позиция 02

**Grouped** (в папках tables/, questions/):
- Формат: `T.P.G.SS.GNN` — Тип.Программа.Руководство.Раздел.НомерВГруппе
- Пример: `Q.1.1.02.05` — Вопрос в программе 1, руководстве 1, разделе 02, позиция 05

### Префиксы типов

| Префикс | Тип объекта |
|---------|-------------|
| T | Таблица (Table) |
| Q | Вопрос (Question) |
| C | Кейс (Case) |

### Физические имена файлов

- Формат: `NN-slug-name.extension`
- Пример: `01-attention/02-focus.md`

## Типы файлов

| Тип | Суффикс | Описание |
|-----|---------|----------|
| Текст | `.md` | Инструкции и теория в Markdown |
| Таблица | `.table.yaml` | Таблица моделирования |
| Кейс | `.case.yaml` | Выбор правильного ответа |
| Вопрос | `.question.yaml` | Открытый ответ |
| Изображение | `.png/.jpg/.svg` | Иллюстрации в папках `assets/` |

## YAML Frontmatter

Каждый `.md` файл должен начинаться с YAML frontmatter:

```yaml
---
id: "1.1.01.02"
title: "Название подраздела"
description: "Краткое описание содержания"
order: 2
guide_ref: "1.1"
section_ref: "1.1.01"
tags: ["внимание", "практика"]
created: "2025-01-15"
updated: "2025-01-15"
---
```

## Шаблоны

В папке `_templates/` доступны шаблоны для создания новых объектов:

| Шаблон | Описание |
|--------|----------|
| `index.md` | Главная страница руководства |
| `subsection.md` | Подраздел с текстом |
| `table.table.yaml` | Таблица моделирования |
| `question.question.yaml` | Открытый вопрос |
| `case.case.yaml` | Кейс с выбором ответа |

## Быстрый старт

1. Склонируйте репозиторий
2. Используйте шаблоны из `_templates/` для новых файлов
3. Следуйте правилам именования и структуры
4. Заполняйте YAML frontmatter для каждого файла
